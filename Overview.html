Goals

  • Define the finite list of encodings for the platform and obsolete the "CHARACTER SETS" registry.
  • Define error handling for those encodings.
  • Converge implementations.
  • Mark encodings that are not utf-8 as legacy.

Notes

  • Labels defined for each encoding is a conservative list at this point. We can extend it as we please provided there is some agreement to do so.
  • Mapping for each encoding is done by majority of implementations. If the majority wants to change or has to change for compatibility, we can change the mapping.
  • The list of encodings is currently incomplete. Not all single-octet encodings are listed. Multi-octet encodings are not listed at all.

1 Terminology

A code point is referenced as a four-or-more digit hexadecimal value preceded by "U+". E.g. U+0020.

An octet is referenced as a double-digit hexadecimal value. E.g. AA.

In a single-octet encoding table the octet can be found by concatenating the row and column header respectively. In these tables code points have "U+" omitted for brevity.

3 Legacy single-octet encodings

An encoding where each octet maps to a code point is an single-octet encoding. encoding defined with subsections of this section are single-octet encodings.

Unless stated otherwise, in an single-octet encoding the octet's hexadecimal value maps to a code point of the same hexadecimal value.

An octet 7E maps to U+007E.

See 8-bit labels and their encodings in various browsers for the research this section is based on.
